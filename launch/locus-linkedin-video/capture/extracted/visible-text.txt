[p] How it works
[h3] 1 · Map the graph
[p] Parse imports into a deterministic dependency graph — routes → components → hooks → libs. No LLM guessing.
[h3] 2 · Localize the task
[p] Match task language against paths and source, then include dependencies and nearby integration points.
[h3] 3 · Widen when uncertain
[p] If no file matches with enough evidence, fall back to the whole repo instead of returning a speculative small slice.
[p] Cross-cutting bugs (a shared util that broke the dashboard) are caught two ways: the shared file is already in the dependency slice, and a recent-change signal floats it to the top — so precision doesn't mean missing the real cause.
